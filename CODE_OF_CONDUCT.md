# Code of Conduct

Sparko is an open-source project for people building practical X workflows with human review, account context, and safe publishing controls. We want the project to be useful, direct, and welcoming.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent, but accept correction when impact does not match intent.
- Keep feedback specific and actionable.
- Help newcomers understand the codebase without making them feel foolish for asking.
- Use fictional data in public examples, screenshots, tests, and docs.
- Discuss product direction in a way that protects user safety, privacy, and trust.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Sexualized language or unwelcome sexual attention.
- Discriminatory comments related to identity, background, ability, religion, nationality, age, or experience level.
- Publishing private information, secrets, credentials, real customer data, or private X account details.
- Pushing spam, engagement bait, deceptive automation, or features that intentionally bypass human review.
- Repeated off-topic disruption after maintainers ask for the thread to return to the project.

## Reporting

If you see behavior that violates this Code of Conduct, contact the maintainer through GitHub. If the report involves a security issue or leaked credentials, follow [SECURITY.md](SECURITY.md) and avoid posting sensitive details publicly.

Reports should include:

- what happened
- where it happened
- who was involved
- whether any private data, credentials, or user safety concerns are involved

## Enforcement

Maintainers may remove comments, close issues, reject pull requests, block users, or take other reasonable steps to protect the project and its community.

Enforcement should be proportional, consistent, and focused on keeping Sparko useful and safe for contributors and users.
